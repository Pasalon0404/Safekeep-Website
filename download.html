manifest.txt contains the SHA-256 hash of the image. manifest.txt.asc is the GPG signature over that manifest. developer-pubkey.asc is the maintainer’s public key. Verification takes two minutes and is the only way to confirm you received an unmodified image.

Don’t Trust, Verify

SafeKeepVault is a bare-metal cryptographic tool. An image tampered with in transit could silently exfiltrate your seed the moment you type it. GPG signature verification and hash checking are not optional steps — they are the security model. Complete all three steps before writing the image to any drive.

Step 01 Import the Developer Key

Download developer-pubkey.asc and import it into your local GPG keyring. You only need to do this once.

Linux / macOS 
# Import the maintainer's public key
$ gpg --import developer-pubkey.asc

GPG is pre-installed on most Linux distributions and macOS. On macOS, install it via brew install gnupg if needed. Windows users can use Gpg4win.

Step 02 Verify the Manifest Signature

Confirm that manifest.txt was signed by the maintainer’s key and has not been modified since signing.

Linux / macOS 
# Verify the signature over the manifest
$ gpg --verify manifest.txt.asc manifest.txt

gpg: Signature made ...
gpg: using RSA key XXXXXXXXXXXXXXXX
gpg: Good signature from "SafeKeepVault Maintainer ..."

You must see Good signature in the output. Any other result — including “BAD signature” or a key mismatch — means the manifest was tampered with. Do not proceed.

Step 03 Verify the Image Hash

Confirm that the image file you downloaded matches the SHA-256 hash recorded in the verified manifest.

Linux 
# Check image hash against the manifest (Linux)
$ sha256sum --check manifest.txt

safekeep-os-v1.0.img: OK
macOS 
# Check image hash against the manifest (macOS)
$ shasum -a 256 -c manifest.txt

safekeep-os-v1.0.img: OK

OK confirms the image is byte-for-byte identical to the file the maintainer signed. Any FAILED result means the download is corrupt or was modified — delete it and download again.

Flash & Boot

Once verified, write the image to a USB drive of 4 GB or larger. The internal hard drive of the host machine is never mounted during a SafeKeepVault session — choose a thumb drive you can dedicate to this purpose. 

# Flash with dd — replace /dev/sdX with your USB device
$ sudo dd if=safekeep-os-v1.0.img of=/dev/sdX bs=4M status=progress oflag=sync

Alternatively, use Balena Etcher for a graphical interface on Linux, macOS, or Windows. Once flashed, configure your machine’s BIOS to boot from USB, insert the drive, and power on. SafeKeepVault loads in under 30 seconds with no network access, no desktop, and no attack surface.
