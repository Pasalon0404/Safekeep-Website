What’s New in v1.1b

Full round-trip compatibility with Nunchuk and Electrum — the two strictest air-gap wallet implementations in the ecosystem. Workflows that previously required workarounds or manual hex conversion now work first-scan, every time.

v1.1b
  • Compat Nunchuk — Raw Hex QR output. The Scribe exports signed PSBTs as un-prefixed Raw Hex QR codes. Nunchuk's strict scanner accepts them instantly. Scan, broadcast, done.
  • Compat Electrum round-trip — Base43 decoder. A custom Base43 decoder is now built into SafeKeep's QR scanner. Electrum's proprietary PSBT encoding is decoded on the fly — the last manual step in the Electrum workflow is gone.
  • New Electrum skeleton wallet export. SafeKeep generates a [FINGERPRINT]-electrum.json with the correct descriptor, derivation path, and ckcc_xfp field. Import it once and your watch-only wallet is ready.

Don’t Trust, Verify

A download that’s been tampered with could steal your seed the moment you type it. Verifying the file takes about two minutes and is the only way to be sure you received exactly what was released. Complete all three steps before writing the image to your USB drive.

Step 01 Import the Developer Key

Download developer-pubkey.asc from the GitHub repository and import it. You only need to do this once.

Linux / macOS 
# Import the maintainer's public key
$ gpg --import developer-pubkey.asc

GPG is pre-installed on most Linux distributions and macOS. On macOS, install it via brew install gnupg if needed. Windows users can use Gpg4win.

Step 02 Verify the Manifest Signature

Confirm that manifest.txt was signed by the maintainer’s key and has not been modified since signing.

Linux / macOS 
# Verify the signature over the manifest
$ gpg --verify manifest.txt.asc manifest.txt

gpg: Signature made ...
gpg: using RSA key XXXXXXXXXXXXXXXX
gpg: Good signature from "SafeKeepVault Maintainer ..."

You must see Good signature in the output. Any other result — including “BAD signature” or a key mismatch — means the manifest was tampered with. Do not proceed.

Step 03 Verify the Image Hash

Confirm that the image file you downloaded matches the SHA-256 hash recorded in the verified manifest.

Linux 
# Check image hash against the manifest (Linux)
$ sha256sum --check manifest.txt

safekeep-os-v1.1b.img: OK
macOS 
# Check image hash against the manifest (macOS)
$ shasum -a 256 -c manifest.txt

safekeep-os-v1.1b.img: OK

OK confirms the image is byte-for-byte identical to the file the maintainer signed. Any FAILED result means the download is corrupt or was modified — delete it and download again.

Flash & Boot

Write the image to a USB drive of 4 GB or larger. Choose a drive you can dedicate to this purpose — the host machine's internal drive is never touched. 

# Flash with dd — replace /dev/sdX with your USB device
$ sudo dd if=safekeep-os-v1.1b.img of=/dev/sdX bs=4M status=progress oflag=sync

Once flashed, set your computer to boot from USB in its startup settings, insert the drive, and power on. SafeKeepVault is ready in under 30 seconds — no internet connection, no distractions, just your keys.
